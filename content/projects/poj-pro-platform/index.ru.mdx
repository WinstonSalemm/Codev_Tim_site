## Overview

Poj Pro Operations Platform — production internal business system для OOO «Poj Pro». Desktop-клиент .NET MAUI подключается к ASP.NET Core API на Railway: продажи, склад, финансы, договоры, KPI менеджеров. Клиент автоматически выбирает online/offline режим и синхронизирует очередь операций из локального SQLite при восстановлении связи. Статус: Production.

Репозиторий: [WinstonSalemm/projectApp](https://github.com/WinstonSalemm/projectApp). Публичный каталог — [`poj-pro-site`](https://www.poj-pro.uz) ([Poj-Pro](https://github.com/WinstonSalemm/Poj-Pro)). Контракты и боты — `poj-pro-api-contracts`, `poj-pro-telegram-bots`.

## Problem Statement

Продажи, склад и финансы жили в таблицах, мессенджерах и разрозненных инструментах. Нужна единая система для наличных/карточных продаж, броней, долгов, поставок ND-40/IM-40, налоговой аналитики (UZ) и договоров — с работой при временной недоступности интернета или Railway API.

## Business Context

Пользователи — внутренние операторы, менеджеры и склад OOO «Poj Pro». Пик нагрузки — рабочие часы по Ташкенту. Desktop — ежедневная операционная surface; публичный сайт https://www.poj-pro.uz обслуживает B2B-покупателей отдельно.

## System Blueprint

Трёхуровневая схема: MAUI client → ASP.NET Core API (Railway) → PostgreSQL + SQLite offline queue на клиенте.

При запуске клиент проверяет доступность API. Online — немедленная запись; offline — очередь с последующей синхронизацией.

## Technology Stack

| Layer      | Technology          | Role                                              |
| ---------- | ------------------- | ------------------------------------------------- |
| Client     | .NET MAUI, .NET 8   | Windows desktop UI, offline queue, sync indicator |
| API        | ASP.NET Core        | Business logic, REST, Railway deploy              |
| Offline    | SQLite              | Локальное хранение и очередь операций             |
| Production | Railway, PostgreSQL | Hosted API и основная БД                          |

## Trade-offs

**Desktop MAUI vs web admin.** Desktop — скорость ввода, offline на точке продаж.

**Monolith API vs microservices.** Один ASP.NET Core deploy упрощает sync и транзакции продаж/склада.

**Отдельный public site repo.** Каталог на poj-pro.uz деплоится независимо от internal ops.

## Roadmap

**Maintained.** Продажи и склад, надёжность offline sync, стабильность Railway API.

**Planned.** Интеграция с `poj-pro-api-contracts` для статусов договоров в клиенте.

**Deferred.** MAUI targets кроме Windows desktop.

## Interface Record

Только Windows desktop. Области: продажи (наличные, карта, Click, бронь, долг), склад и поставки, клиенты и должники, договоры, финансы и кассы, налоговая аналитика, KPI менеджеров. Индикатор sync — правый верхний угол.

## Lessons Recorded

**Очередь sync должна быть видна.** Счётчик pending операций повышает доверие операторов.

**Public web и internal ops — разные продукты.** Разделение на `Poj-Pro` и `projectApp` снизило regression risk.

**API URL — конфиг окружения.** Base URL Railway в `appsettings.json` клиента.
