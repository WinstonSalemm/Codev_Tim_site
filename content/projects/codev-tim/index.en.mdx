## Overview

Codev_Tim is the public engineering operating system representing software engineer Timur — a modular Next.js 15 site with product registry, trilingual MDX content pipeline, terminal-style navigation, and contact workflows. Status: In Development at v0.9.4.

Repository: [WinstonSalemm/Codev_Tim_site](https://github.com/WinstonSalemm/Codev_Tim_site) (public). This record describes the same codebase that powers the live site deployment on Railway.

## Problem Statement

Generic portfolio templates hide product architecture, registry context, and engineering decisions. Codev_Tim needs a dedicated platform surface where each registered product has an Engineering Record, the registry is first-class navigation, and content stays trilingual without duplicating site logic per locale.

## Business Context

Primary audience: technical buyers, engineering leads, and collaborators evaluating Timur's systems. Traffic is documentation-heavy — registry drill-down, project records, about and contact paths. The site is the canonical public face of the Codev_Tim brand, not a slide deck or link tree.

## System Blueprint

App Router application with locale-prefixed routes, server-side content loaders, and client modules for dashboard, registry, and terminal UX.

## Trade-offs

**Engineering OS vs classic portfolio.** Chosen to expose registry, architecture docs, and product proof — not only screenshots.

**MDX in repo vs headless CMS.** File-based content keeps version control and build-time validation; CMS deferred until editorial volume justifies it.

**Meta product in own registry.** `codev-tim` slug documents this repository explicitly so the platform is auditable like any other product.

## Roadmap

**Maintained.** Product registry accuracy, trilingual records, theme and onboarding UX, contact delivery.

**Planned.** Confirmed production `SITE_URL`, expanded Knowledge Base cross-links.

**Deferred.** Authenticated operator modules beyond public contact flow.

## Interface Record

Public browser interface: Operations Center (dashboard), Product Registry, per-project Engineering Records, About, Principles, Writing, Contact. Terminal module provides in-site command navigation. Locales: `/en`, `/ru`, `/uz`.

## Lessons Recorded

**Registry slugs must match public repos.** Drift between metadata and GitHub undermines trust for technical readers.

**This site documents itself.** Keeping `codev-tim` in the registry forces the same Engineering Record standard as client products.

**Build-time search index.** Generated `search-index.json` keeps registry and records discoverable without a separate search service.
