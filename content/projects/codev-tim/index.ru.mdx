## Overview

Codev_Tim — public engineering operating system, представляющая software engineer Timur. Модульный сайт на Next.js 15: product registry, трёхъязычный MDX pipeline, terminal-style navigation и contact workflows. Статус: In Development, v0.9.4.

Репозиторий: [WinstonSalemm/Codev_Tim_site](https://github.com/WinstonSalemm/Codev_Tim_site) (public). Эта запись описывает тот же codebase, который деплоится на Railway.

## Problem Statement

Типовые portfolio templates скрывают product architecture и engineering decisions. Codev_Tim нужен как dedicated platform surface: Engineering Record на каждый продукт, registry как first-class navigation, trilingual content без дублирования логики по locale.

## Business Context

Аудитория — technical buyers, engineering leads, collaborators. Трафик documentation-heavy: registry drill-down, project records, about и contact. Сайт — canonical public face бренда Codev_Tim.

## System Blueprint

App Router с locale-prefixed routes, server-side content loaders и client modules для dashboard, registry и terminal UX.

## Trade-offs

**Engineering OS vs portfolio.** Registry и architecture docs важнее screenshots.

**MDX in repo vs CMS.** File-based content + build-time validation; CMS отложен.

**Meta product в registry.** Slug `codev-tim` документирует этот repository явно.

## Roadmap

**Maintained.** Точность registry, trilingual records, theme/onboarding, contact.

**Planned.** Подтверждённый production `SITE_URL`, Knowledge Base cross-links.

**Deferred.** Authenticated operator modules.

## Interface Record

Public browser: Operations Center, Product Registry, Engineering Records, About, Principles, Writing, Contact. Terminal module — command navigation. Locales: `/en`, `/ru`, `/uz`.

## Lessons Recorded

**Registry slugs должны совпадать с public repos.** Расхождение с GitHub снижает доверие.

**Сайт документирует себя.** `codev-tim` в registry — тот же стандарт Engineering Record.

**Build-time search index.** `search-index.json` без отдельного search service.
